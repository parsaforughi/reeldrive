# پرداخت آنلاین Pro با بله‌پی (از طریق WooCommerce)

هدف: یه فروشگاه ووکامرس کوچیک که فقط یه کارش خرید اشتراک Proه. ربات لینک چک‌اوت رو با `telegram_id` و `days` می‌سازه، ووکامرس بعد از پرداخت یه webhook به داشبورد ربات می‌زنه، ربات خودکار Pro رو فعال می‌کنه و به کاربر پیام می‌ده.

هیچ تماسی با API خام بله‌پی لازم نیست — افزونه بله‌پی خودش گیت‌وی رو مدیریت می‌کنه، ما فقط به رویداد «سفارش تکمیل شد» ووکامرس گوش می‌دیم.

## ۱. بالا آوردن وردپرس روی Railway

توی Railway پروژه: **New → Template → WordPress** (یا از Railway Marketplace یه تمپلیت WordPress+MySQL انتخاب کن). بعد از دیپلوی، یه دامنه/ساب‌دامنه بهش وصل کن (مثلاً `shop.reeldrive.app` یا یه ساب‌دامین دیگه).

## ۲. نصب‌ها

داخل وردپرس:

1. **ووکامرس** رو نصب و فعال کن (پلاگین رایگان از مخزن وردپرس).
2. **افزونه بله‌پی** (فایل zip که خریدی) رو از Plugins → Add New → Upload Plugin نصب کن، فعالش کن، تنظیماتش (merchant id و مشابه) رو طبق راهنمای خودِ افزونه پر کن.
3. افزونه‌ی رایگان **Code Snippets** رو نصب کن (برای اضافه‌کردن کد PHP بدون دست‌زدن به قالب).

## ۳. ساخت محصولات

برای هر کدوم از ۵ پلن، یه محصول ساده (Simple product) بساز با قیمت دقیقاً برابر تومنی که تو ربات تعریف شده (`pro_toman_monthly` × ماه):

| مدت | محصول |
|---|---|
| ۳۰ روز | Pro ۱ ماهه |
| ۶۰ روز | Pro ۲ ماهه |
| ۹۰ روز | Pro ۳ ماهه |
| ۱۸۰ روز | Pro ۶ ماهه |
| ۳۶۵ روز | Pro ۱ ساله |

بعد از ساخت هر محصول، **آیدی محصول** (عدد کنار لینک ویرایش، مثلاً `post=123`) رو یادداشت کن — لازمشون داری.

## ۴. اسنیپت PHP (گرفتن telegram_id/days)

از فایل `docs/snippets/woocommerce-telegram-meta.php` همین ریپو کپی کن و بذار تو Code Snippets → Add New → پیستش کن → Save and Activate.

این اسنیپت `telegram_id` و `days` رو از URL می‌گیره و روی سفارش ذخیره می‌کنه — همون چیزی که webhook ما می‌خونه.

(تنظیمات ووکامرس → General → Redirect to cart بهتره **خاموش** باشه تا کاربر مستقیم بره چک‌اوت، ولی خاموش نبودنش هم مشکلی نداره.)

## ۵. Webhook ووکامرس

WooCommerce → Settings → Advanced → Webhooks → Add webhook:

- **Name:** Reeldrive Pro
- **Status:** Active
- **Topic:** Order updated
- **Delivery URL:** `https://<دامنه-داشبورد-ربات>/api/webhook/woocommerce`
- **Secret:** یه رشته‌ی رندوم طولانی بساز (مثلاً با `openssl rand -hex 32`) — این دقیقاً باید با `WOOCOMMERCE_WEBHOOK_SECRET` که تو Railway ست می‌کنی یکی باشه.

## ۶. متغیرهای Railway (سرویس ربات/داشبورد)

```
WOOCOMMERCE_BASE_URL=https://shop.reeldrive.app
WOOCOMMERCE_WEBHOOK_SECRET=همون رشته‌ی مرحله ۵
WOOCOMMERCE_PRODUCT_IDS=30:ID1,60:ID2,90:ID3,180:ID4,365:ID5
```

با آیدی محصولاتی که تو مرحله ۳ یادداشت کردی. بعد از ست‌کردن، Redeploy.

وقتی این سه متغیر پر باشن، گزینه‌ی «پرداخت آنلاین (بله‌پی)» خودکار تو شاپ مینی‌اپ تلگرام (`/subscribe`) ظاهر می‌شه.

## ۷. تست

یه سفارش تستی واقعی (کمترین مبلغ) از تلگرام بزن، پرداخت رو کامل کن، لاگ Railway رو نگاه کن — باید ببینی:

```
{"ok": true, "granted": true, "telegram_id": ..., "days": ...}
```

و تو تلگرام پیام «Pro فعال شد» بیاد. اگه چیزی لاگ شد با `"skipped": ...` یعنی یه‌جای زنجیره (meta، وضعیت سفارش، یا امضای webhook) درست تنظیم نشده — پیام skip دقیقاً می‌گه مشکل از کجاست.

## نکات امنیتی

- Secret رو جایی commit نکن — فقط تو Railway env.
- وضعیت‌های قابل قبول برای فعال‌سازی از env قابل تنظیمه: `WOOCOMMERCE_PAID_STATUSES` (پیش‌فرض `completed,processing`) — اگه افزونه بله‌پی وضعیت دیگه‌ای استفاده می‌کنه (مثلاً `bale-pay-paid`) این متغیر رو عوض کن.
- هر سفارش فقط یک‌بار پردازش می‌شه (idempotent) — حتی اگه ووکامرس webhook رو چندبار بفرسته، Pro تکراری اضافه نمی‌شه.
